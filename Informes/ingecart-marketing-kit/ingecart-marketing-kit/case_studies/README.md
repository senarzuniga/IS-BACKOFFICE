# Ingecart — Case Studies

This directory stores documented customer success stories and use cases
that can be used in sales presentations, proposals, and the Adaptative-Sales-Engine.

> **Note:** Case studies are populated as pilot projects with Ingecart customers
> are completed and consent to publish is obtained.

## Case Study Template

Each case study follows this structure:

```
case_study_CLIENTCODE_TOPIC.md
```

### Required Sections

1. **Customer Context** — Industry, size, challenge
2. **Solution Deployed** — Which IAR agents, integration approach
3. **Implementation Timeline** — Phases and duration
4. **Results** — Quantified metrics (downtime reduction %, quote speed, incidents)
5. **Customer Quote** — Anonymised or attributed testimonial
6. **ROI Summary** — Investment vs. annual benefit

## Usage by Adaptative-Sales-Engine

Case studies are indexed in Supabase under `ingecart_documents`
with `category = 'case_study'`. The Sales Engine can retrieve them to:
- Match a prospect's profile to relevant case studies
- Extract ROI data points for proposal personalisation
- Identify objection-handling evidence
